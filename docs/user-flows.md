## 2. UF-001: 文档上传与向量化

### 2.1 主流程（Happy Path）

```mermaid
flowchart TD
    A[用户进入应用] --> B{文档库是否为空?}
    B -- 是 --> C[显示空状态引导]
    B -- 否 --> D[显示文档列表]
    C --> E[用户点击/拖拽上传]
    D --> E
    E --> F[选择文件]
    F --> G{校验文件类型}
    G -- 通过 --> H{校验文件大小}
    G -- 不通过 --> G1[提示: 不支持的文件类型]
    H -- 通过 --> I{校验文档总数}
    H -- 不通过 --> H1[提示: 文件超过 20MB]
    I -- 通过 --> J[上传文件到服务器]
    I -- 不通过 --> I1[提示: 文档数已达上限 100]
    J --> K[解析文档为文本]
    K --> L{解析字数为 0?}
    L -- 是 --> L1[提示: 疑似扫描件,暂不支持]
    L -- 否 --> M[递归切分文本]
    M --> N[调用 Embedding 生成向量]
    N --> O[存入 Chroma 向量库]
    O --> P[保存元数据到 MariaDB]
    P --> Q[更新文档列表状态为完成]
    Q --> R[显示: 上传成功,切分为 N 个片段]
```

### 2.2 状态流转

| 步骤 | 状态标签 | 视觉反馈 | 预计耗时 |
|------|----------|----------|----------|
| 上传文件 | 上传中（黄色） | 进度条 + "上传中" | 1-5s |
| 解析文档 | 解析中（黄色） | 旋转图标 + "解析中" | 1-3s |
| 文本切分 | 切分中（黄色） | 旋转图标 + "切分中" | < 1s |
| Embedding | 向量化中（黄色） | 旋转图标 + "向量化中" | 2-10s/文件 |
| 存储 | 向量化中（黄色） | 同上 | < 1s |
| 完成 | 完成（绿色） | 绿色标签 | - |

### 2.3 异常流程

| 异常场景 | 触发条件 | 处理方式 | 用户看到 |
|----------|----------|----------|----------|
| 不支持的文件类型 | 扩展名不在 [pdf,docx,md,txt] 内 | 前端拦截 | Toast: "不支持的文件类型，请上传 PDF/Word/Markdown/TXT" |
| 文件过大 | 单文件 > 20MB | 前端拦截 | Toast: "文件超过 20MB 限制" |
| 文档数超限 | 已有 ≥ 100 个文档 | 后端拒绝 | Toast: "文档数已达上限 100，请先删除不需要的文档" |
| 扫描件 PDF | 解析后字数 = 0 | 后端返回特定错误 | 文档列表状态"失败" + 提示"疑似扫描件，MVP 暂不支持 OCR" |
| 解析失败 | PyPDF2/python-docx 异常 | 后端捕获 | 状态"失败" + "文档解析失败，请检查文件是否损坏" |
| Embedding 失败 | OpenAI API 错误/网络超时 | 后端重试 1 次 | 状态"失败" + "向量化失败，请检查 API Key 或网络" + 重试按钮 |
| 网络中断 | 上传中断 | 前端检测 | Toast: "网络中断，请重试" |

### 2.4 批量上传流程

```mermaid
flowchart LR
    A[选择多个文件] --> B[逐个校验]
    B --> C[并行上传]
    C --> D[各自独立处理]
    D --> E[列表显示各自状态]
```

**说明**：批量上传时每个文件独立处理，互不影响。前端显示每个文件的独立状态。后端可并行处理（并发受限于 Embedding API 速率）。

## 3. UF-002: RAG 问答

### 3.1 主流程（Happy Path）

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant BE as 后端
    participant V as 向量库
    participant LLM as 大模型
    U->>FE: 输入问题
    FE->>FE: 校验非空
    FE->>FE: 显示用户消息气泡
    FE->>FE: 显示"检索中"加载态
    FE->>BE: POST /api/chat/messages (SSE)
    BE->>BE: 问题向量化
    BE->>V: 检索 top_k=5
    V-->>BE: 相关片段 + 元数据
    BE->>BE: 过滤相似度 < 0.3
    BE->>BE: 组装上下文(系统提示+RAG+历史4轮+问题)
    BE-->>FE: event: references (引用来源)
    FE->>FE: 渲染引用来源卡片
    BE->>LLM: stream=True
    loop 流式生成
        LLM-->>BE: token
        BE-->>FE: event: token
        FE->>FE: 逐字追加到回答区
    end
    BE-->>FE: event: done
    BE->>BE: 保存消息到 MariaDB
    FE->>FE: 移除光标,显示完整回答
    FE->>FE: 输入框清空,恢复可用
```

### 3.2 交互状态

| 阶段 | 视觉状态 | 用户可操作 |
|------|----------|------------|
| 输入问题 | 输入框聚焦，发送按钮可用 | 输入文字 |
| 发送中 | 用户消息出现，输入框禁用 | 等待 |
| 检索中 | "检索中"加载态（对话区） | 可点击"停止" |
| 引用到达 | 引用卡片渲染（回答上方） | 可展开/折叠引用 |
| 首 token 到达 | 光标闪烁，逐字显示 | 可点击"停止生成" |
| 流式中 | 逐字追加 | 可点击"停止生成" |
| 完成 | 光标消失，回答完整 | 可输入新问题 |
| 错误 | 红色错误提示 + 重试 | 可点击"重试" |

### 3.3 多轮对话流程

```mermaid
flowchart TD
    A[用户提问] --> B[RAG 检索 + LLM 生成]
    B --> C[保存问答到历史]
    C --> D{用户继续提问?}
    D -- 是 --> E[加载最近 4 轮 8 条历史]
    E --> F[组装上下文: 系统提示 + RAG + 历史 + 新问题]
    F --> B
    D -- 否 --> G[结束]
```

**上下文组装顺序**（DEC-011）：
1. 系统提示（~200 token）：定义 AI 角色与回答规范
2. RAG 上下文（top_k=5 × 500 字符）：当前问题检索到的文档片段
3. 历史对话（最近 4 轮 8 条消息）：保持多轮连贯性
4. 当前问题：用户本次提问

**截断规则**：
- 超过 4 轮时，最早的轮次不进入上下文
- Token 总预算 ~6000，超出时优先保留系统提示与 RAG 上下文

### 3.4 异常流程

| 异常场景 | 触发条件 | 处理方式 | 用户看到 |
|----------|----------|----------|----------|
| 空问题 | 输入框为空点发送 | 前端拦截 | 发送按钮禁用 |
| 无相关内容 | 检索结果全部低于阈值 0.3 | 不调用 LLM | 提示卡片："未在文档库中找到相关内容，请尝试换个问题或上传更多文档" |
| LLM 超时 | 调用 > 30s | 后端中断 | event: error → "回答生成超时，请重试" + 重试按钮 |
| LLM 错误 | API Key 无效/额度不足 | 后端捕获 | event: error → "回答生成失败：{原因}" + 重试 |
| 网络中断 | SSE 连接断开 | 前端检测 | "网络中断，已生成的部分内容已保存" |
| 文档库为空提问 | 无文档时提问 | 后端检查 | 提示："知识库为空，请先上传文档" |
| 会话历史加载失败 | DB 查询失败 | 后端降级 | 使用空历史继续（不影响当前问答） |

### 3.5 停止生成流程

```mermaid
flowchart TD
    A[流式回答中] --> B[用户点击 停止生成]
    B --> C[前端 abort fetch 请求]
    C --> D[后端检测连接断开]
    D --> E[停止 LLM stream]
    E --> F[保存已生成部分到 DB]
    F --> G[回答区显示已生成内容]
    G --> H[标注: 回答已停止]
```

## 4. UF-003: 会话管理

### 4.1 新建会话

```mermaid
flowchart TD
    A[用户点击 + 新建会话] --> B[前端创建空会话]
    B --> C[POST /api/chat/sessions]
    C --> D[后端保存到 MariaDB]
    D --> E[返回 session_id]
    E --> F[会话列表顶部添加新项]
    F --> G[当前会话切换为新会话]
    G --> H[对话区显示空状态: 开始你的第一个提问]
```

### 4.2 切换会话

```mermaid
flowchart TD
    A[用户点击会话列表项] --> B[GET /api/chat/sessions/{id}/messages]
    B --> C[后端查询历史消息]
    C --> D[返回消息列表]
    D --> E[对话区渲染历史问答]
    E --> F[当前会话高亮]
    F --> G[输入框聚焦,可继续提问]
```

### 4.3 删除会话

```mermaid
flowchart TD
    A[用户点击删除按钮] --> B[弹出确认对话框]
    B --> C{用户确认?}
    C -- 取消 --> D[关闭对话框]
    C -- 确认 --> E[DELETE /api/chat/sessions/{id}]
    E --> F[后端级联删除会话+消息]
    F --> G[列表移除该项]
    G --> H{删除的是当前会话?}
    H -- 是 --> I[切换到第一个会话或空状态]
    H -- 否 --> J[保持当前会话]
```

### 4.4 清空所有会话

```mermaid
flowchart TD
    A[用户点击清空所有] --> B[弹出确认对话框]
    B --> C{用户确认?}
    C -- 取消 --> D[关闭对话框]
    C -- 确认 --> E[DELETE /api/chat/sessions]
    E --> F[后端删除所有会话+消息]
    F --> G[列表清空]
    G --> H[对话区显示空状态]
```

### 4.5 历史持久化（刷新恢复）

```mermaid
flowchart TD
    A[用户刷新页面] --> B[前端加载]
    B --> C[GET /api/chat/sessions]
    C --> D[返回会话列表按时间倒序]
    D --> E[默认选中第一个会话]
    E --> F[GET /api/chat/sessions/{id}/messages]
    F --> G[渲染历史消息]
    G --> H[页面恢复到刷新前状态]
```

## 5. UF-004: 文档管理

### 5.1 查看文档列表

```mermaid
flowchart TD
    A[用户进入文档列表] --> B[GET /api/documents?page=1&size=20]
```

```mermaid
flowchart TD
    A[用户点击删除] --> B[确认对话框]

    C -- 取消 --> D[关闭]

    F --> H[返回成功]

    H --> I[列表移除该项]

```

## 6. UF-005: Embedding Provider 切换

### 6.1 切换流程

```mermaid
flowchart TD
    A[用户点击 Provider 切换] --> B[弹出确认对话框]
    B --> C["提示: 切换方案需重建索引"]
    C --> D{用户确认?}
    D -- 取消 --> E[关闭对话框]
    D -- 确认 --> F[POST /api/config/embedding-provider]
    F --> G[后端切换 provider]
    G --> H[遍历所有文档重新向量化]
    H --> I[进度条显示重建进度]
    I --> J[完成后 Toast: 切换成功]
    J --> K[顶部显示新 provider 名称]
```

### 6.2 异常流程

| 异常场景 | 处理方式 |
|----------|----------|
| 本地模型未下载 | 提示"本地模型未下载，正在下载 2.3GB，请耐心等待" + 下载进度 |
| 重建失败 | 回滚到原 provider + "切换失败，已恢复原方案" |
